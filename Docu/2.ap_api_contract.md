# Registration

## Endpoint
POST /registry/register

## AP request body
Use the same shape as robots for now:

```json
{
  "mac": "aa:bb:cc:dd:ee:ff",
  "ip": "192.168.1.10",
  "scanner_version": "apBundle1.0",
  "capabilities": "ap,status,traffic,association,txpower"
}
```

### Notes:
  Keep scanner_version field name for now to reuse existing NMS and common registration code.

  capabilities tells NMS this is an AP endpoint, not a robot.

## NMS response
Same shape as current robots:

```json
{
  "scanner": "ap-lab-east-01",
  "llm_weblink": "https://chatgpt.com/",
  "time": "2026-03-10-18:30:00",
  "time_format": "%Y-%m-%d-%H:%M:%S"
}
```

### Notes:
  Reuse scanner field name for now.

  AP will store it into scanner_name.txt just like robots.

# 10-second status + command-poll channel
I recommend one new AP-specific API rather than overloading robot poll too much.

## Endpoint
POST /ap/poll/{scanner}

### Why POST, not GET:

  status payload is structured and larger than simple query params

  association table belongs in JSON body

  cleaner than stuffing everything into query args

## AP request body
```json
{
  "time": "2026-03-10-18:30:10",
  "status": {
    "mac": "aa:bb:cc:dd:ee:ff",
    "ip": "192.168.1.10",
    "ssids": ["Lab-5G", "Lab-2G"],
    "band": "5g",
    "channel": 44,
    "antenna_count": 2,
    "associations": [
      {
        "sta_mac": "11:22:33:44:55:66",
        "ssid": "Lab-5G",
        "mcs": 7
      },
      {
        "sta_mac": "22:33:44:55:66:77",
        "ssid": "Lab-5G",
        "mcs": 5
      }
    ]
  }
}
```

## NMS response body
```json
{
  "scanner": "ap-lab-east-01",
  "server_now": "2026-03-10-18:30:10",
  "time_format": "%Y-%m-%d-%H:%M:%S",
  "commands": [
    [
      "1773133049125-0",
      {
        "cmd_id": "1773133049125-0",
        "category": "ap",
        "action": "ap.association.get",
        "execute_at": "2026-03-10-18:30:10",
        "created_at": "2026-03-10-18:30:05",
        "args_json": "{}"
      }
    ]
  ]
}
```

### Notes:
  Keep the command queue item shape identical to robots.

  Only the endpoint is new; command envelope stays the same.

# 60-second traffic report

## Endpoint
POST /ap/traffic/{scanner}

This should only be called when enabled.

## AP request body
```json
{
  "time_start": "2026-03-10-18:29:00",
  "time_end": "2026-03-10-18:30:00",
  "records": [
    {
      "sta_mac": "11:22:33:44:55:66",
      "ac": "BE",
      "avg_frame_duration_us": 820.5,
      "frame_count": 1200,
      "mcs_distribution": {
        "0": 15,
        "1": 30,
        "2": 100,
        "3": 180,
        "4": 250,
        "5": 300,
        "6": 220,
        "7": 105
      }
    },
    {
      "sta_mac": "11:22:33:44:55:66",
      "ac": "VO",
      "avg_frame_duration_us": 410.2,
      "frame_count": 120,
      "mcs_distribution": {
        "5": 20,
        "6": 60,
        "7": 40

```

### Notes:

  strictly per STA, per AC

  no useless AP-wide aggregates

  mcs_distribution should be counts, not percentages
